Executable Constraint Factors for Wikidata Repair

A self-contained research overview of the project idea, scientific motivation, model suite, current results, revised paper narrative, and next experiments.

1. Executive Summary

Abstract. Collaborative knowledge graphs such as Wikidata use soft property constraints to flag suspicious or inconsistent statements, yet repairing these violations remains difficult: a locally plausible edit may imitate historical curator behavior while introducing or preserving other constraint violations. We study neural repair under this fidelity-safety tension. Given a violated Wikidata statement, local graph context, and historical repair supervision, the task is to predict an add/delete edit that both resembles curator behavior and improves the post-edit symbolic state. We introduce executable constraint factors, a neuro-symbolic graph representation in which locally applicable constraints are represented as typed factor nodes. These factors estimate constraint satisfaction and inject role-conditioned pressure messages into a graph neural repair model. Experiments on a stratified Wikidata repair benchmark show that executable factors substantially improve historical edit prediction, raising micro-F1 from 0.5434 to 0.6785 over a passive constraint-context baseline (my ESWC paper's model). However, this improvement does not automatically yield safer repairs: the factorized imitation model worsens aggregate local satisfaction and secondary-regression metrics. A global-only reranking reference achieves the best local satisfaction, but it does so by converging to a delete-focus policy that removes the violating statement and sharply sacrifices historical fidelity. This result reveals a central degeneracy in global symbolic optimization: post-edit satisfaction can be improved by deleting evidence rather than by producing semantically useful repairs. These findings suggest that learned knowledge graph repair should evaluate curator imitation and symbolic safety as separate objectives, and that safe repair requires decision-level objectives that preserve repair intent while controlling secondary constraint effects.

Keywords: knowledge graph repair; Wikidata constraints; neuro-symbolic learning; executable constraint factors; graph neural networks; edit imitation; symbolic safety.

2. Scientific Problem

Wikidata is a collaborative knowledge graph with soft, evolving constraints. A constraint violation indicates that a statement is suspicious or inconsistent with a property-level rule, but not every violation should be repaired in the same way. Human curators often fix one local issue at a time, and their edits reflect conventions, local context, and practical trade-offs rather than global logical optimization.

Neural repair systems can learn from historical edit logs. Given a violation instance, they predict an add/delete edit that resembles a human repair. This is useful, but it creates a narrow optimization target: the model can become good at imitating the historical edit while remaining unaware of how that edit affects other locally applicable constraints.

Historical Fidelity

Does the model predict the same repair slots that curators historically used? This is measured with precision, recall, and micro-F1 over edit slots.

Symbolic Safety

Does the predicted edit fix the primary violation and avoid making other local constraints worse? This requires symbolic post-edit evaluation.

The central research question is whether models can improve both at once, or whether these two goals form a fidelity-safety frontier.

3. Research Idea

The method can be read as a neuro-symbolic repair model. The neural part learns from historical repair examples and predicts edit slots. The symbolic part evaluates what those edits do to Wikidata constraints. The proposed bridge between the two is to expose symbolic constraint structure inside the neural graph representation.

Terminology used in this document

Several terms are specific to this project, so they are introduced here before the model descriptions:

  • A violation instance is one observed constraint violation together with its local graph context and the historical curator edit used as supervision.
  • The primary constraint is the violated constraint that defines the repair task for that instance.
  • Secondary constraints are other locally applicable constraints that may be affected by the same edit.
  • A candidate repair is one possible add/delete edit that can be evaluated symbolically before choosing it.
  • A reranker is a second-stage model that receives candidate repairs and selects one according to an objective such as global constraint satisfaction.

Executable constraint factors

The core idea is to represent locally applicable constraints as executable factor nodes inside each violation subgraph. A factor node is a learned module associated with one local constraint. It is typed by a constraint family, such as single, conflictWith, type, valueType, or inverse. Each factor is connected to the local variables that determine its satisfaction.

Instead of being a passive label or connector, a factor node computes a differentiable approximation of constraint satisfaction and emits messages back to scoped variable nodes. These messages are called constraint pressure. Intuitively, pressure is the model's learned signal that some nearby entity, property, or value is relevant to satisfying or violating a constraint. The graph neural network can use these pressure signals while it builds representations for repair prediction.

Relation to factor-graph neural models

The architecture is related to neural factor-graph message-passing models, but it should not be framed as a new general-purpose Factor Graph Neural Network. Existing models such as Zhang et al.'s Factor Graph Neural Network, Satorras and Welling's Neural Enhanced Belief Propagation, and Kuck et al.'s Belief Propagation Neural Network already show that factor-graph structure can be neuralized for approximate inference and representation learning. They use variable-to-factor and factor-to-variable message passing to estimate marginals, refine belief-propagation messages, decode structured variables, or approximate partition functions.

Related model Commonality with this project Key difference
Zhang et al. FGNN Uses variable nodes, factor nodes, and learned variable-factor message passing to model higher-order dependencies. Targets general factor-graph representation/inference; this project uses factors as typed Wikidata constraint instances for repair prediction.
Satorras and Welling NEBP Combines factor-graph GNN messages with belief-propagation-style inference. Refines BP messages for probabilistic inference tasks; this project uses neural factors to inject constraint pressure into a KG repair model.
Kuck et al. BPNN Preserves factor-graph structure and learns BP-like iterative operators. Focuses on partition-function/model-count estimation and BP-like guarantees; this project focuses on local graph edits, curator fidelity, and secondary no-regression.

The scientific contribution is therefore not the generic use of factor nodes. The contribution is to make symbolic KG constraints executable inside a learned repair model. Each local constraint instance becomes a typed operator that estimates satisfaction and sends role-conditioned pressure messages to the variables in its scope, while symbolic validators remain responsible for grounded candidate-level safety evaluation. In this sense, the model uses the factor-graph abstraction not to approximate belief propagation, but to make soft Wikidata constraints active participants in safer local edit prediction.

Repair representation

The repair target is a six-slot add/delete edit: subject_add, predicate_add, object_add, subject_delete, predicate_delete, and object_delete. Historical repair fidelity is evaluated by comparing predicted slots to the curator edit.

Primary and secondary constraints

Each instance has a primary violated constraint. The local subgraph can also contain secondary constraints that may be affected by the predicted edit. The project therefore distinguishes:

  • Primary repair: whether the originally violated constraint is fixed.
  • Secondary no-regression: whether other satisfied constraints remain satisfied.
  • Global local satisfaction: the aggregate post-edit state of all local constraints.

The important distinction is that a model can use these factor signals for different purposes. It can use them to imitate the historical edit more accurately, or it can use them to select an edit that improves symbolic constraint satisfaction. The current results show that the first effect is strong, while the second requires more explicit decision-level machinery.

4. Methodology in Equations

This section gives a compact mathematical view of the method. The equations are intentionally written at the level of the paper idea rather than repository internals.

Repair instance and local constraints

Each training example starts from a graph state before repair, a historical edit, and a set of locally applicable constraints. One of those constraints is the primary violation; the rest are secondary constraints that might be affected by the same edit.

Repair instance \[ \begin{aligned} x &= \left(G^{-}, \Delta^{\star}\right), C_{\mathrm{local}} \\ C_{\mathrm{local}} &= \{c^{\star}, c_1, \ldots, c_m\} \end{aligned} \]

Here, G- is the pre-repair local graph, Delta* is the historical curator edit, C_local is the set of local constraints, and c* is the primary violated constraint.

Six-slot repair target

The model predicts a single add/delete repair in six slots: three slots for a triple to add and three slots for a triple to delete.

Repair edit \[ \Delta \left(s^{+}, p^{+}, o^{+}, s^{-}, p^{-}, o^{-}\right) \]

Executable factor representation

For every local constraint instance c, the model builds a typed factor node. Its scope identifies the variable nodes needed to evaluate that constraint: predicates, subjects, objects or values, depending on the constraint family. The factor aggregates those scoped embeddings into a local constraint state, then uses a type-specific executor to estimate whether the current configuration is satisfied.

Factor execution \[ \begin{aligned} z_c^{(k)} &= \operatorname{AGG}_{t(c)} \left(\{h_v^{(k)} : v \in \operatorname{Scope}(c)\}\right) \\ \hat{s}_c^{(k)} &= \sigma\!\left(f_{t(c)}\!\left(z_c^{(k)}\right)\right) \\ \operatorname{viol}_c^{(k)} &= 1 - \hat{s}_c^{(k)} \\ m_{c \to v}^{(k)} &= g_{t(c), r(v,c)} \!\left( z_c^{(k)}, h_v^{(k)}, \operatorname{viol}_c^{(k)} \right) \end{aligned} \]

t(c) is the constraint family, and r(v,c) is the role of variable v inside that constraint, such as predicate, subject, object, or value. Parameters are shared by family, so all single factors share one executor, all type factors share another, and so on. This is the compositional part of the model: new local graphs reuse the same learned constraint operators across many concrete constraint instances.

The violation signal may be passed as an explicit scalar, as written above, or carried implicitly in the learned factor state produced by f_t. In both cases, the factor is not a hardcoded symbolic validator; it is a differentiable approximation trained from factor satisfaction labels and repair behavior.

Message passing with constraint pressure

Variable embeddings are updated in two stages. First, ordinary graph message passing collects neighborhood context from the data graph. Then scoped constraint factors inject pressure back into the variables they touch. This second term is how local constraint state becomes visible to the repair decoder without hardcoding a delete-or-add rule.

Variable update \[ h_v^{(k+1)} = \operatorname{GNN}\!\left(h_v^{(k)}, \mathcal{N}(v)\right) + \lambda \sum_{c:\, v \in \operatorname{Scope}(c)} m_{c \to v}^{(k)} \]

The coefficient lambda controls the strength of the residual pressure update. Multiple factors can act on the same variable, so the model can represent competing local constraint signals rather than treating each violation in isolation. The pressure message is role-conditioned: a factor can send different information to the predicate than to the subject or value node.

The H2 pressure-masking diagnostics test whether this pathway matters by removing all pressure, keeping only primary-factor pressure, or keeping only secondary-factor pressure at inference time.

5. Model Suite and Training Objectives

The project compares a small sequence of models. The names are shorthand for positions in the research comparison: start with a passive learned baseline, add executable factor structure, then test whether candidate-level safety objectives can use that structure to make safer repair choices. The global-only reranker is included as an endpoint: it shows what happens when post-edit symbolic satisfaction is optimized directly.

Training objectives

The common learned objective is edit imitation: predict the historical repair slots. Factorized models also learn auxiliary factor satisfaction predictions.

Imitation and factor supervision \[ \begin{aligned} \mathcal{L}_{\mathrm{edit}} &= \sum_{j=1}^{6} \operatorname{CE}\!\left( y_j^{\star}, P\!\left(y_j \mid G^{-}, C_{\mathrm{local}}\right) \right) \\ \mathcal{L}_{\mathrm{factor}} &= \sum_{c \in C_{\mathrm{local}}} \left[ \operatorname{BCE}\!\left(s_c^{-}, \hat{s}_c^{\mathrm{pre}}\right) + \operatorname{BCE}\!\left(s_c^{+}, \hat{s}_c^{\mathrm{post}}\right) \right] \end{aligned} \]

The practical model variants add different safety terms. M1C scores symbolic candidate repairs with a chooser loss. M1D adds direct expected penalties for primary failure and secondary regression. G0 instead optimizes expected post-edit global satisfaction. The empirical result is that this objective converges to deleting the focus statement, which is why G0 is interpreted as the global-satisfaction/delete-focus endpoint rather than as an independently useful repair policy.

Model objectives \[ \begin{aligned} \mathcal{L}_{\mathrm{A1}} &= \mathcal{L}_{\mathrm{edit}} + \eta\,\mathcal{L}_{\mathrm{factor}} \\ \mathcal{L}_{\mathrm{M1C}} &= \mathcal{L}_{\mathrm{edit}} + \eta\,\mathcal{L}_{\mathrm{factor}} + \mu\,\mathcal{L}_{\mathrm{chooser}} \\ \mathcal{L}_{\mathrm{M1D}} &= \mathcal{L}_{\mathrm{edit}} + \eta\,\mathcal{L}_{\mathrm{factor}} + \alpha\,\mathcal{L}_{\mathrm{primary}} + \beta\,\mathcal{L}_{\mathrm{secondary}} \\ \mathcal{L}_{\mathrm{G0}} &= -\mathbb{E}_{\Delta \sim \pi_{\theta}} \left[ \frac{1}{|C_{\mathrm{local}}|} \sum_{c \in C_{\mathrm{local}}} S_c\!\left(G^{\Delta}\right) \right] \end{aligned} \]

6.Evaluation

For evaluation, the predicted edit is applied to the pre-repair graph and the constraint set is evaluated symbolically. Let S_c(G) be 1 if constraint c is satisfied in graph state G, and 0 otherwise.

Post-edit graph \[ \begin{aligned} G^{\Delta} &= \operatorname{Apply}\!\left(G^{-}, \Delta\right) \\ S_c(G) &\in \{0, 1\} \end{aligned} \]

The central safety metrics are computed over the local constraints:

Safety metrics \[ \begin{aligned} \operatorname{PrimaryFix} &= S_{c^{\star}}\!\left(G^{\Delta}\right) \\ \operatorname{GFR} &= \frac{1}{|C_{\mathrm{local}}|} \sum_{c \in C_{\mathrm{local}}} S_c\!\left(G^{\Delta}\right) \\ \operatorname{SRR} &= \frac{ \left|\{c \neq c^{\star} : S_c(G^{-}) = 1 \land S_c(G^{\Delta}) = 0\}\right| }{ \left|\{c \neq c^{\star} : S_c(G^{-}) = 1\}\right| } \\ \operatorname{SIR} &= \frac{ \left|\{c \neq c^{\star} : S_c(G^{-}) = 0 \land S_c(G^{\Delta}) = 1\}\right| }{ \left|\{c \neq c^{\star} : S_c(G^{-}) = 0\}\right| } \end{aligned} \]

These equations make the central result easier to interpret. Micro-F1 measures whether Delta resembles the historical edit Delta*. The safety metrics measure what Delta does to the symbolic constraint state after it is applied. The project shows that those two evaluations do not necessarily agree.

Sort:
Metric Meaning Interpretation
Micro-F1 ↑ Historical edit-slot fidelity. Higher means better imitation of curator edits.
Primary fix rate ↑ Whether the predicted edit fixes the original violation. Higher means better intent fulfillment.
GFR ↑ Global fix rate over the local constraint set after applying the edit. Higher means better local symbolic satisfaction.
SIR ↑ Secondary improvement rate. Higher means more previously unsatisfied secondary constraints become satisfied.
SRR ↓ Secondary regression rate. Lower means fewer previously satisfied secondary constraints are broken.
Disruption ↓ Mean number of edit operations or changed triples. Lower usually means more edit-minimal repairs.

The paper should not rank models by a single metric. A model can be better at historical imitation while being worse at symbolic no-regression.

The goal is not to declare a single winner, but to see which part of the research hypothesis each model supports: historical imitation, primary repair, secondary no-regression, or global local consistency. Its selected edits are overwhelmingly delete-focus edits, so its high satisfaction scores should be read as satisfaction-by-evidence-removal rather than as a successful general repair strategy. Supported: representation helps imitation

A1 improves micro-F1 from 0.5434 to 0.6785 relative to B0. This is the cleanest positive result for executable factors.

Not supported: representation alone improves safety

A1 is worse than B0 on aggregate GFR, SRR, and SIR. The factorized representation does not automatically reduce secondary regressions.

Mixed: local safety objectives

The completed M1D is directionally promising, especially in dense slices, but the completed aggregate result is not enough to claim broad baseline dominance. The completed non-zero-primary M1C follow-up does not improve aggregate SRR in this table and also lowers fidelity and primary-fix behavior.

Finding: global-only optimization degenerates

G0 improves global satisfaction by converging to a delete-focus policy. Its match with H1 exposes a satisfaction-by-deletion failure mode rather than a practical repair strategy.

8. H2 Diagnostics

The main result table tells us what happened, but not why it happened. H2 is the diagnostic layer designed to answer that question. It is run only for factorized models because it inspects internal factor outputs and pressure pathways that do not exist in the passive B0 baseline.

H2 has three purposes. First, it checks whether the learned factor modules can predict symbolic constraint satisfaction. Second, it slices performance by local constraint density, because a repair with many nearby constraints is a harder and more relevant case for this project. Third, it masks parts of the factor-pressure mechanism at inference time to test whether the model is actually using those signals.

Factor semantics

In this diagnostic, the model's factor heads are treated as satisfaction predictors. A high score means that the model has learned to approximate whether a local constraint is satisfied before repair or after the historical repair. This does not by itself prove that the final edit choice is safe, but it tells us whether the constraint modules contain useful information.

Pressure masking

Pressure masking is an intervention: the model is evaluated again while some factor-pressure messages are removed. If performance changes substantially, the removed pathway was causally important for prediction. If performance barely changes, the model was not relying much on that pathway.

10. Scientific Relevance

The project is scientifically relevant because it challenges a common assumption in data-driven repair: that learning to imitate historical fixes is enough. In collaborative knowledge graphs, historical repairs are useful supervision, but they are not a complete proxy for post-edit consistency.

For Knowledge Graph repair

The work introduces a richer evaluation surface that measures collateral constraint effects, not only whether the primary violation was repaired.

For neuro-symbolic learning

The results show that symbolic factors can be learned and used by neural models, but that using them for safe decisions requires more than representation learning.

For benchmark design

The paper separates historical fidelity from graph safety, making failure modes visible that would be hidden by edit-slot F1 alone.

For future systems

The results point toward constrained reranking, candidate validity checks, calibration, and stronger secondary-pressure routing as concrete next steps.
